# RecycleMap СПб — MVP (дизайн-документ)

**Дата:** 2026-04-21
**Автор:** Дмитрий (первый программный проект, новичок)
**Статус:** утверждён к реализации

## 1. Цель и контекст

Учебный MVP-аналог [recyclemap.ru](https://recyclemap.ru) — интерактивная карта пунктов приёма вторсырья для Санкт-Петербурга. Цель — освоить стек (Next.js + TypeScript + Leaflet) на реальном, полезном продукте, а не на абстрактном туториале.

Собственный визуальный стиль (не клонировать оригинал), но та же базовая идея: карта + фильтры по категориям сбоку.

## 2. Аудитория

Любой житель Петербурга, который хочет сдать вторсырьё и ищет ближайшую точку. Мобильный трафик ожидается — дизайн должен нормально работать на телефоне.

## 3. Объём MVP

### Входит (Definition of Done)

- Карта Санкт-Петербурга с 25–30 реальными точками приёма
- Цветные маркеры (цвет по первой категории точки)
- Клик по маркеру → попап с деталями (адрес, часы, телефон, сайт, описание, бэйджики категорий)
- Левая панель фильтра с 8 чекбоксами категорий (эмодзи + название + цветовая метка)
- Фильтр работает по ИЛИ: точка видна, если хотя бы одна её категория выбрана
- Счётчик "Найдено: N точек"
- Кнопки "Выбрать все" / "Сбросить"
- Адаптив: на мобильном фильтры в выдвижной шторке снизу
- Страница `/about` с кратким описанием проекта
- Задеплоено на Vercel с публичным URL

### Не входит (отложено)

- Поиск по адресу
- Геолокация пользователя ("найти рядом со мной")
- Текстовый список точек рядом с картой
- Форма "предложить точку"
- Модерация / админка
- Аутентификация
- Кластеризация маркеров (для 30 точек не нужна)
- Настоящая БД
- Мультиязычность
- Рейтинги, отзывы, фото

## 4. Стек

- **Next.js 15** (App Router) + **TypeScript**
- **Tailwind CSS** — стили
- **react-leaflet + Leaflet** — карта
- **OpenStreetMap** — тайлы (бесплатно)
- **Хранилище данных:** статичный `data/points.json` в репозитории
- **Деплой:** Vercel через GitHub

### Почему такой выбор

- **JSON вместо БД** — минимум отвлечений от главного (карта + фильтры). Редактирование данных = `git push`. Для учебного MVP это плюс — подопечный попутно осваивает git.
- **Leaflet вместо MapLibre** — проще API, больше туториалов, для 30 маркеров разницы в производительности нет.
- **App Router** — современный стандарт Next.js 15, новички всё равно будут учить его.
- **Эмодзи-иконки категорий** — в MVP быстро; на SVG перейдём позже, если захочется эстетики.

  "address": "Средний пр. В.О., 45",
  "lat": 59.9429,
  "lng": 30.2772,
  "categories": ["plastic", "glass", "paper", "metal"],
  "hours": "Пн–Пт 10:00–19:00, Сб 11:00–17:00",

### TypeScript-типы (`lib/types.ts`)

```ts

  | 'plastic' | 'glass' | 'paper' | 'metal'

│ ☑ 📦 Тетрапак  │     • попап при клике              │
│ ☑ 👕 Текстиль  │                                    │

│ Выбрать все    │                                    │
│ Найдено: 18    │                                    │
└────────────┴────────────────────────────────────────┘
```

- Шапка: узкая, слева название "RecycleMap СПб", справа ссылка на `/about`.
- Левая колонка фильтра — около 260px.
- Карта — оставшееся пространство. Начальная центровка на центре Питера (примерно 59.9386, 30.3141), zoom ~11.

### Мобильный

- Карта на весь экран.
- Фильтры — выдвижная шторка снизу (bottom-sheet), вызывается кнопкой "Фильтры".
- Счётчик точек — на кнопке фильтров: "Фильтры (18)".

### Поведение фильтра

- Все 8 категорий включены по умолчанию.
- Логика: `point.categories.some(c => selected.has(c))`.
- Переключение чекбокса мгновенно обновляет видимость маркеров.
- "Сбросить" — снимает все галочки (карта пустая, счётчик 0).
- "Выбрать все" — ставит все галочки.

### Попап маркера

Содержимое при клике по маркеру:

- Название (заголовок)
- Адрес
- Строка бэйджиков: эмодзи + название категории, цвет фоном (все категории точки)
- Часы работы (если есть)
- Телефон (ссылка `tel:`)
- Сайт (ссылка, открывается в новой вкладке)
- Описание
- (Поле `source` в UI не показываем — оно для меня, для проверки данных)

Пустые поля просто не рендерятся.

## 7. Структура проекта

```
recyclemap/
├── app/
│   ├── layout.tsx          # общая обёртка (шапка, метаданные)
│   ├── page.tsx            # главная: фильтры + карта
│   ├── about/page.tsx      # страница "О проекте"
│   └── globals.css         # Tailwind + Leaflet CSS
├── components/
│   ├── Header.tsx
│   ├── FilterPanel.tsx
│   ├── Map.tsx             # обёртка с dynamic import
│   ├── MapInner.tsx        # Leaflet, рендерится только на клиенте
│   └── PointPopup.tsx
├── data/
│   ├── points.json
│   └── categories.json
├── lib/
│   └── types.ts
├── public/                 # (иконки если перейдём на SVG)
├── package.json
├── next.config.ts
├── tsconfig.json
└── tailwind.config.ts
```

### Потоки данных

- `app/page.tsx` (client component) импортирует `points.json` и `categories.json`, держит `useState<Set<CategoryId>>` выбранных фильтров, вычисляет отфильтрованный список и передаёт его вниз.
- `FilterPanel` принимает `categories`, `selected`, `onToggle`, `onSelectAll`, `onReset`, `pointCount`.
- `Map` (обёртка) делает `dynamic(() => import('./MapInner'), { ssr: false })`.
- `MapInner` рисует `<MapContainer>`, `<TileLayer>` (OSM), и `<Marker>` для каждой точки. Попап — `<PointPopup point={...} categories={...} />`.

### Одна "хитрая" часть — SSR и Leaflet

Leaflet обращается к `window` при инициализации, но Next.js рендерит компоненты и на сервере, где `window` нет. Решение — всегда импортировать `MapInner` через `next/dynamic` с `ssr: false`. В документе фиксируем это как обязательное требование.

## 8. Сбор данных

- Я (ассистент) нахожу 25–30 точек в СПб через открытые источники (карты города, группы ВК экодвижений, сайты пунктов приёма).
- Для каждой точки сохраняю `source` URL, откуда взяты данные.
- Пользователь проверяет список перед мержем.
- Координаты получаем геокодингом адресов (через Nominatim или вручную из Яндекс/Google Maps).

## 9. Деплой

- GitHub-репозиторий `recyclemap-spb`.
- Vercel-проект, подключённый к репозиторию — автодеплой при пуше в `main`.
- Публичный URL на поддомене `.vercel.app`.
- Кастомный домен — опционально, после MVP.

## 10. Не-цели и принятые ограничения

- **Код не покрывается тестами.** Для учебного MVP без бизнес-логики тесты — лишний слой сложности. Когда появится БД или формы ввода, вернёмся к этому.
- **Доступность (a11y) — базовая:** семантический HTML, labels у чекбоксов. Полный аудит — не в этом спринте.
- **Производительность — не оптимизируем.** 30 маркеров работают быстро "из коробки".
- **SEO — минимум:** корректный `<title>`, `description`. OG-теги, sitemap — потом.

## 11. Процесс работы и темп

Пользователь — новичок, никогда не писал код. Реализация будет идти пошагово:

1. Установка инструментов (Node.js, VS Code, git, аккаунт GitHub) — займёт больше времени, чем кажется.
2. Создание Next.js проекта и настройка Tailwind.
3. Сбор данных в `points.json`, `categories.json`.
4. Шапка и раскладка страницы.
5. Панель фильтров + состояние.
6. Карта, маркеры, попап.
7. Адаптив (мобильная шторка).
8. Страница `/about`.
9. Деплой на Vercel.

Темп диктует пользователь. После каждого крупного шага проверяем, что всё работает локально.

## 12. Риски и чем закрыть

| Риск | Вероятность | Чем закроем |
|---|---|---|
| Ошибка SSR при Leaflet | высокая | явный `dynamic import` с `ssr: false`, зафиксирован в архитектуре |
| Плохая геолокация точек | средняя | источник (`source`) для каждой точки, пользователь проверит вручную |
| Мобильная шторка будет глючить | средняя | возьмём готовую библиотеку (vaul) вместо самописной |
| Застревание на установке инструментов | высокая | в плане отдельный шаг "окружение", без перехода дальше пока `npm run dev` не запустится |
| Тайлы OSM перестанут грузиться | низкая | OSM бесплатный, но с лимитом; для 1 пользователя ок, при росте трафика — MapTiler/Stadia |
