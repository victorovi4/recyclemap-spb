## Обновления данных через чат-бот

**Идея:** публика и волонтёры сообщают об изменениях в точках (часы поменялись, точка закрылась, появилась новая) через мессенджер, а не через сайт.

**Варианты платформы:**
- **Max-bot** (боты на платформе VK Messenger / Max) — нативный для российской аудитории
- **VK Mini App** — внутри ВКонтакте, можно прикручивать как web-view с ограниченным UI
- **Telegram-бот** — самая простая DX, но политически спорно в РФ
- **WhatsApp Business API** — если хочется широкого охвата

**MVP-функциональность бота:**
1. `/new_point` — отправить новую точку (адрес текстом или геопозиция → категории чекбоксами → фото → комментарий)
2. `/update_point` — выбрать точку поблизости → изменить часы / отметить "закрыто" / добавить фото
3. Все сообщения идут в очередь модерации в нашей админке
4. Модератор принимает/отклоняет, боту приходит уведомление "ваше предложение принято"

**Технически:**
- Отдельный сервис (Next.js API route или отдельный node-процесс)
- Webhook от мессенджера → парсинг команды → insert в таблицу `submissions`
- Та же очередь модерации что и для сабмитов с сайта

**Когда делать:** после того, как текущая итерация ("клон СПб с БД + админка") будет в проде и станет понятно, что поток апдейтов от пользователей реально нужен.
